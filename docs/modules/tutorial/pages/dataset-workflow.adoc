== 1. Discover Available Data Sources

[source,bash,subs=attributes+]
----
kub-dataset list-dmps
kub-dataset list-locations --show-versions
----

If you need a specific backend only:

[source,bash,subs=attributes+]
----
kub-dataset list-locations --dmp girder-unistra --show-versions
----

== 2. Pull Location Dataset

[tabs]
====
Public Dataset::
+
--
[source,bash,subs=attributes+]
----
kub-dataset pull {tutorial_location} \
  --version {tutorial_version} \
  --cemdb-root {tutorial_cemdb_locations} \
  --dmp girder-unistra
----
--

Authenticated Pull::
+
--
[source,bash,subs=attributes+]
----
kub-dataset pull {tutorial_location} \
  --version {tutorial_version} \
  --cemdb-root {tutorial_cemdb_locations} \
  --dmp girder-unistra \
  --api-key "$GIRDER_API_KEY"
----
--
====

[NOTE]
====
Current `kub-dataset` defaults use split archive behavior for `pull` and `push` when `--type all` is used.
====

== 3. Pull Simulator Dataset

[source,bash,subs=attributes+]
----
kub-dataset pull-simulator \
  --version {tutorial_simulator_version} \
  --cemdb-root {tutorial_cemdb_root} \
  --force
----

== 4. Verify Local Dataset State

[source,bash,subs=attributes+]
----
kub-dataset summary {tutorial_location} --cemdb-root {tutorial_cemdb_locations}
kub-dataset manifest-show {tutorial_location} --version {tutorial_version} --cemdb-root {tutorial_cemdb_locations}
----

== 5. Optional: Track Pulls With DVC

[source,bash,subs=attributes+]
----
kub-dataset init
kub-dataset pull {tutorial_location} --version {tutorial_version} --dvc --force --cemdb-root {tutorial_cemdb_locations}
kub-dataset status
----

== Expected Layout Check

[source,bash,subs=attributes+]
----
find {tutorial_cemdb_locations}/{tutorial_location}/v{tutorial_version} -maxdepth 2 -type d | sort
----

You should see key directories such as `geo/`, `weather/`, `scenarios/`, `preprocessing/`, and `simulations/`.
